深蓝学院：移动机器人路径规划

第一章、概述与课程介绍

1.1 课程介绍

  • 学习路径规划的前置技能：线性代数、概率论、MATLAB、CPP…….

  • 自主机器人的组成：状态估计(定位)、感知、规划、控制

  • 规划路径的要求：安全(无碰撞)、光滑、运动学

  • 前端：路径规划， 后端：轨迹生成(轨迹优化)

    在低维状态空间，通常是离散的，去寻找一条初始的、安全的、让机器人可以通行的、解的质量不是特别好的初始值，我们称他为路径，path。这个路径通常是没有时间信息的，只局限于低维的、没有高维信息的。之后我们通过后端的轨迹优化，把这个低维的path，转变为或者说在他周围搜索一条高维的可以执行的轨迹(Trajectory)，这通常在一个高维的连续的状态空间中进行的，要用到很多的优化技巧。

1.2 运动规划算法分类

  • 前端路径规划：

    • 基于搜索(A星、D、D星、跳点)
    • 基于采样(概率路图、RRT、RRT星….)
    • 基于满足运动学要求的路径规划(混合A星….)

  • 后端轨迹优化(轨迹生成)

    • min snap
    • 在硬约束和软约束下的min snap轨迹优化方法

  • 马尔科夫决策过程

  • MPC模型预测控制

1.3 常用地图结构

  • 占用栅格地图(稠密、结构化)
  • 八叉树地图(来源于八叉树这一数据结构、细分障碍物区域、可以调节地图的精细程度、非稠密、结构化)
  • Voxel hashing(类似于八叉树地图、非结构化、哈希键值查询地图、主要用于三维重建)
  • Point cloud map
  • TSDF map (截断的符号距离场)
  • ESDF map (不截断的欧式符号距离场)

第二章、基于搜索的路径规划

2.1 图搜索基础

  1. 图(graph)由节点和边组成，边可以赋予权重和方向。

  2. 图搜索的一般步骤：

    1. 起始点start_node 加入 openList
    2. 扩展start_node，扩展节点加入openList
    3. pop出openList中cost最小的节点，判断是否到达，没有则加入closedList
    4. 扩展pop出的节点，并加入openList中
    5. 循环4、5，直至找到重点

  3. 广度优先搜索：新进新出，维护的是一个队列；深度优先搜索：新进后出，维护的是一个堆栈；

2.2 Dijkstra和A星算法

2.2.1 Dijkstra算法

  • D算法维护一个优先级队列
  • D算法的cost是起点到当前点的距离；cost_g;
  • D算法对OpenList中的节点会更新他们的cost，同时更新他们的父节点

2.2.2 A星算法

  • 贪星算法 + Dijkstra = A星；
  • Heuristic 启发性的；
  • A星Cost = cost_g + cost_h；其中cost_h为当前点到终点的距离；
  • A星最优行的保证前提是：cost_h小于该点到终点的真实距离；
  • 权重A星，cost_h_weighted = weightd * cost_h。weightd > 1,该值越大，算法越接近谈心算法，会带来速度的提升，但是没有办法保证最优性。
  • 在空旷场景应该使A算法趋近于贪心该算法；

第三章、基于采样的路径规划

3.1 RRT*

  • 相较RRT增加了chooseParent()和rewire()环节。该改进措施保证了RRT星的渐进最优性；

3.2 kinodynamic-RRT*

  • 考虑机器人的运动学约束；
  • 使用曲线代替直线；

3.3 Anytime-RRT*

  • 实时更新路径，一直采样；
  • 适用于环境实时变化的场景；

3.4 Informed-RRT*

  • 构建椭圆形非均匀采样区域；

3.5 Cross-entropy montion planing

  • 对初始路径中的点，每个点构建一个圆形采样区域，重复采样；

3.6 总结

关于RRT算法的改进主要集中在各个环节。

第四章、运动学约束下的路径规划

4.1 动力学概念简介

  • Kinodynamic : Kinematic + Dynamic 该词是一个合成词（运动学+动力学）
  • 为什么要在前端路径规划时考虑运动学约束？
    • 机器人不能够走折线
    • 后端的轨迹优化，在前端路径太差的情况下，优化的效果不会太理想

4.2 状态栅格搜索算法

  • 满足运动学约束的路径规划关键在于：连接节点的边要是feasible motion connections
  • 为了实现这种特殊的边，有两种方法
    • 正向方法：离散控制空间，机器人以不同的速度前进的轨迹。给定U、T，计算S（状态）
      • 缺点：控制空间的离散缺乏目标导向
    • 反向方法： 离散状态空间，对于两个确定的状态，求解中间的路径(边)。给定S（状态），计算U、T
      • 优点：有目标的导向性
      • 缺点：求解比较困难

1.1.5 线性回归模型

概念：训练集（数据集）、训练符号

1.1.6 代价函数

$$
J = \frac{1}{2m}*\sum_{i=1}^{m}({y_{prediction}-y_{true}})^2
$$

m是样本数量，$ y_{prediction}$是模型预测值，$y_{true}$是训练集中的真值，$J$为总代价(代价函数)

1.1.7 梯度下降

对于线性回归模型可以表示为
$$
y = wx + b
$$
带入代价函数可以得到
$$
J = \frac{1}{2m}\sum_{i=1}^{m}{(wx+b-y_{true})^2}
$$
计算$ J$关于$w$的偏导数
$$
\frac{\partial J}{\partial w} = \frac{1}{2m} \frac{\partial}{\partial w} \sum_{i=1}^{m}{(wx+b-y_{true})^2}= \frac{1}{m}\sum_{i=1}^{m}{(wx+b-y_{true})}x
$$
计算$J$关于$b$的偏导数
$$
\frac{\partial J}{\partial b} = \frac{1}{2m}\frac{\partial}{\partial b}\sum_{i=1}^{m}{(wx+b-y_{true})^2}=\frac{1}{m}\sum_{i=1}^{m}{(w*x+b-y_{true})}
$$
梯度下降的具体步骤为
$$
w=w-\alpha *\frac{\partial J}{\partial w} \
b=b-\alpha *\frac{\partial J}{\partial b}
$$
其中$\alpha$为学习率，控制梯度下降的速度，也控制着学习的速率。

在使用dynamic_reconfigure::Client时出一个奇怪的现象，getCurrentConfiguration()获取不到现有的参数。查看该函数的实现

该函数接收参数ConfigType& configuration和const ros::Duration& timeout = ros::Duration(0))前者是函数的返回值：存贮读取到的参数；后者是可配置参数，默认值为ros::Duration(0)，默认情况是一直读取直到读取成功，在手动设置超时阈值后会按照超时计算。

调节这个超时参数发现没有效果，getCurrentConfiguration()一直返回false，而setConfiguration函数可以改变参数的值。导致原有参数被清空，仅有设置后的参数为正常值。

解决方法：在实例化EB_dyn_client_之后，等待几秒ros::Duration(3.0).sleep()。

RRT_start_DEBUG 2025-03-07|DEBUG|DEBUG

在写RRT*算法时出现在将CloseList转换成path时，程序出现死循环，原因是三个节点互为父节点，其中一个是终点。

出现这个的原因是中间在_checkGoal()函数中将目标点加入CloseList时没有更新其代价值。

在goal_point加入到CloseList后，RRT*的chooseParent环节中new_point重新选择父节点时遍历到新加入的goal_point，由于goal_point的代价值为0，此时new_point会选择goal_point做为父节点。

在之后的循环中goal_point的父节点有概率选择new_point作为父节点（此时new_point的代价很低），形成闭环。

“An Overview of Nature-Inspired, Conventional, and Hybrid Methods of Autonomous Vehicle Path Planning”

文章把路径规划算法分为三类：受自然启发的、传统的(非自然)、混合；

1 传统方法

传统方法主要依靠与障碍物的距离、排斥与吸引力、统计方法、聚类或图形地图等信息，进行路径规划。

1.1 人工势场法(APF)

规划目标吸引车辆、障碍物排斥车辆。改进方法：考虑家养动物的运动属性扩展吸引力和排斥力的计算，改善人机交互；势场窗口。

1.2 基于视觉的路径规划

该方法的核心在于利用图像信息提取障碍物，并设计规避障碍物的算法。

1.3 沿墙规划(WF)

利用两个相邻的红外传感器实现。缺点是容易忽略微小障碍物。

1.4 滑膜控制规划(SMC)

计算很昂贵。

1.6 动态窗口法(DWA)

容易陷入局部最优。

1.7 A*

1.8 VD Planner

在具体的工实践中不容易实现

链表ListNode

链表(单向)中一个节点会记录下一个节点的地址，这样的优势是一个链表不需要连续的内存空间。

在做题时碰到一段代码tail = tail->next，看到这段的时候我以为tail->next替换了tail。但实际上并没有。

在官方题解中tail首先被赋予了一个值，然后又经历了上面的流程。

代码第一行：将第二个节点赋值

代码第二行：将第一个节点变成第二个节点。这么做并不会导致链表断开。程序中头节点用head表示，且head = tail。这样第一个节点是可以被找到的，第二个节点现在是tail，现在考虑第三个节点：

首先第三个节点被赋值tail->next = new ListNode(sum % 10)，然后将tail变成第三个节点。但是！但是！但是！第二个节点并没有消失！第二个节点还可以通过head->next访问到。

这是因为tail在这里只是个临时变量。在将tail变成第三个节点的过程中tail = tail->next,第二个节点所在的地址的内容并没有发生变化，即head->next没有发生变化。在一次次的tail=tail->next的过程中，改变的只是tail这个临时容器。
