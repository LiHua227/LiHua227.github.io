CloudCompare_点云匹配
2025-03-21|DEBUG
Word count: 186|Reading time: 1 min

在现场的建筑场景，环境经常变化，需要频繁更换地图。新地图的原点要求和原地图的原点对齐。

具体实现流程是：将两张地图放到CloudCompare中进行匹配，得到一个变换4维变换矩阵。对新地图施加这个变换，并保存。

问题在于：两张地图在CloudCompare中对齐之后，发布到RVIZ中发现并没有对齐。如下图:

查找原因发现，点云的数据结构中有一个属性叫viewpoint，经过CloudCompare保存后的PCD地图这个值被修改了。

默认的viewpoint为 0 0 0 1 0 0 0；

修改这个值和原地图相同，在RVIZ中的效果如下，问题解决。
